# electron-cloud-simulanion

Простой учебный проект на C++ для моделирования динамики электронов в электростатическом поле. Программа рассчитывает движение набора электронов с учётом их взаимного кулоновского взаимодействия и воздействия внешнего источника поля — неподвижного протона.

## О проекте

В текущей реализации моделируется система из **64 электронов** и **1 протона**. Протон расположен в начале координат и создаёт внешнее электрическое поле. Электроны также взаимодействуют друг с другом. На каждом шаге рассчитываются силы, после чего положение и скорость электронов обновляются по простой явной схеме интегрирования.

Проект оформлен как консольное приложение **Visual Studio / MSVC** и может использоваться как базовая заготовка для:

- учебных экспериментов по численному моделированию;
- изучения электростатики и движения заряженных частиц;
- дальнейшего развития в сторону визуализации траекторий;
- тестирования собственных методов интегрирования.

## Реализованная модель

### Частицы

- **Электрон**
  - масса: `m_e = 1`
  - заряд: `q_e = -1`
- **Протон**
  - заряд: `q_p = 1`

### Электрическое поле

Поле от частицы рассчитывается в виде

```text
E(r) = q * Δr / |Δr|^3
```

где `Δr` — вектор от источника поля к точке наблюдения.

### Интегрирование движения

Для каждого электрона выполняется обновление состояния:

```text
r(t + dt) = r(t) + v(t) * dt
v(t + dt) = v(t) + F(t) * dt / m
```

В проекте используется простая явная схема, близкая к методу Эйлера. Она удобна для демонстрационных задач, но для более устойчивого и точного моделирования в дальнейшем лучше перейти на Leapfrog, Verlet или Runge–Kutta.

## Начальные условия

В коде заданы следующие параметры:

- количество электронов: `N_e = 64`
- количество протонов: `N_p = 1`
- число шагов моделирования: `t_limit = 100`
- шаг по времени: `dt = 0.1`
- координаты протона: `(0, 0, 0)`
- начальная скорость всех электронов: `(0, 1, 0)`

Начальные координаты электронов формируются на плоскости `z = 0` в виде сетки `8 × 8`.

### Назначение файлов

- `electron dynamics.cpp` — точка входа, инициализация системы, основной цикл моделирования, запись результатов.
- `electron.h` — класс электрона: положение, скорость, обновление состояния, расчёт поля.
- `proton.h` — класс протона и расчёт его поля.
- `vector.cpp` — реализация простого 3D-вектора и базовых операций над ним.
- `data.txt` — файл, в который сохраняются результаты симуляции.
- `electron dynamics.sln` / `.vcxproj` — файлы решения и проекта Visual Studio.

## Формат выходных данных

Результаты работы программы записываются в `data.txt`.

Файл имеет следующую структуру:

1. Первая строка — число временных шагов `t_limit`
2. Вторая строка — число электронов `N_e`
3. Далее для каждого шага:
   - `N_e` строк с состояниями электронов;
   - строка-разделитель:

```text
new step
```

Каждая строка электрона содержит:

```text
x    y    z    vx    vy    vz
```

где координаты и компоненты скорости разделены символами табуляции.

## Сборка и запуск

### Требования

- Windows
- Visual Studio 2022
- MSVC toolset v143

### Как запустить

1. Откройте файл `electron dynamics.sln` в Visual Studio.
2. Выберите нужную конфигурацию, например `Debug | x64`.
3. Соберите проект.
4. Запустите приложение.
5. После выполнения в каталоге проекта будет создан или обновлён файл `data.txt`.

## Ограничения текущей реализации

Проект рабочий как учебный прототип, но в нём есть ряд упрощений:

- используются безразмерные единицы;
- отсутствует обработка особенностей при очень малом расстоянии между частицами;
- расчёт выполняется по схеме `O(N²)` по числу электронов;
- нет графической визуализации траекторий;
- файл `vector.cpp` фактически используется как заголовочный файл, что лучше переработать в `vector.h` + `vector.cpp`.
